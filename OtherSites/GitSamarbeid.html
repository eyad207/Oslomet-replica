Slik har vi brukt GitHub for å samarbeide

Alle gruppemedlemmene har sin egen branch hvor de gjør sine egne endringer. Oppgaver ble fordelt mellom de forskjellige medlemene. f,eks. Fylle main, lage header og footer, lage de de andre nettsidene, som denne siden for eksempel. Når en person følte at de hadde lagt til en god endring viste de den fram til resten av gruppa, og gruppa som en helhet godkjente den. Små endringer trengte ikke bli godkjent.

Det hendte også iblant at endringene våre førte til konflikter når vi prøvde å merge den med main branch. Da satte vi opp endringene på måten vist i bildet (vi kaller det for «diff-visning»), også diskuterer vi rundt hvilken av versjonene som skal beholdes på innenfor ulike seksjoner av koden. Deretter løste vi de konfliktene og merga de med main, og syncet de med github slik at de andre kunne se dem.

Fordeler med GitHub

  • GitHub gir oss muligheten til å danne, redigere og oppdatere et prosjekt sammen på kun en fil
  • Ved hjelp av branches, kan vi gjøre regelmessige endringer uten å grykte for at noe går galt med opplasting eller endlastingen av filen som fører til at vi mister arbeid
  • Det er også veldig mye enklere å synkronisere prosjektet med github enn det hadde vært å måtte gjøre det manuelt med konseptet merging av branches kombinert med at man synkronisere github med VS-code
