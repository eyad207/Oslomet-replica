Slik har vi brukt GitHub for å samarbeide

Alle gruppemedlemmene har sin egen branch hvor de gjør sine egne endringer. Oppgaver ble fordelt mellom de forskjellige medlemene. f,eks. Fylle main, lage header og footer, lage de de andre nettsidene, som denne siden for eksempel.

Når en person følte at de hadde lagt til en god endring viste de den fram til resten av gruppa, og gruppa som en helhet godkjente den. Små endringer trengte ikke bli godkjent.

Fordeler med GitHub

GitHub gir oss muligheten til å danne, redigere og oppdatere et prosjekt sammen på kun en fil. Vi kan gjøre regelmessige endrigner uten å frykte for at noe går galt med opplastning eller nedlastningen av filen som fører til at vi mister arbeid. Det er også veldig mye enklere å synkronisere prosjektet med github enn det hadde vært å måtte gjøre det manuelt. I bildene under kan du se eksempler på hvordan vi har brukt GitHub til å gjøre felles endringer
